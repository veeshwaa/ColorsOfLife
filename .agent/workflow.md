# WORKFLOW.md

# AI Collaboration Workflow

Version:
1.0

Purpose:

This document defines **how** every AI agent collaborates on this repository.

It does not define the story.

It does not define the author's style.

It defines the working process.

---

# Core Principle

Never assume.

Never invent.

Never silently modify.

When uncertainty exists,

**stop and ask the author.**

---

# Repository Read Order

Every new session must begin in the following order.

1. AGENT.md
2. AUTHOR.md
3. STYLE.md
4. STORY.md
5. SAMPLE.md

Never begin writing before understanding all five documents.

---

# Before Every Scene

Before writing any scene:

* Read the previous completed scene.
* Read the future dependency (if already planned).
* Read STORY.md for continuity.
* Understand where this scene begins.
* Understand where this scene must end.
* Identify:

  * Characters
  * Timeline
  * Location
  * Important objects
  * Emotional objective
  * Future dependencies

If anything is uncertain,

**stop and ask.**

---

# Scene Development Process

Never begin writing immediately.

Follow this order.

### Step 1 — Purpose

Why does this scene exist?

If removed, what breaks?

---

### Step 2 — Emotion

How should readers feel while reading?

How should they feel after finishing?

---

### Step 3 — Continuity

What previous scene created this one?

What future scene depends upon it?

---

### Step 4 — Writing

Only now begin writing.

Never optimize while writing.

First preserve the author's intent.

Only then improve clarity.

---

# After Every Scene

Verify:

✓ chronology

✓ character consistency

✓ object continuity

✓ emotional progression

✓ suspense

✓ memorable object

✓ future setup

✓ continuity with previous scenes

✓ no contradiction with future locked scenes

If any item fails,

rewrite the scene.

---

# Character Rules

Characters never change personality for plot convenience.

Characters make decisions.

The plot follows naturally.

Never reverse this relationship.

---

# Dialogue Rules

Dialogue should reveal people.

Not explain information.

Avoid exposition.

If both characters already know something,

they should not explain it merely for the reader's benefit.

---

# Mystery Rules

Reveal information naturally.

Never reveal too early.

Never hide information artificially.

Readers should assemble the story themselves.

Curiosity is stronger than surprise.

---

# Revision Rules

Locked scenes are not rewritten unless:

* continuity is broken
* factual contradiction exists
* author explicitly approves the change

Grammar improvements alone are not sufficient.

---

# New Ideas

Every new idea belongs to one category.

### Immediate

Needed for the current scene.

Implement immediately.

---

### Future

Useful later.

Record it.

Do not interrupt the current chapter.

---

### Parking Lot

Interesting but unrelated.

Store separately.

Never derail the current writing session.

---

# Continuity Rules

Always track:

* character ages
* dates
* locations
* relationships
* object continuity
* knowledge possessed by each character
* unresolved mysteries
* future foreshadowing

Future knowledge must never leak into earlier chapters.

---

# Before Committing Any Scene

Ask one final question.

> **Could Biswa naturally write this?**

If the answer is uncertain,

rewrite.

If the answer is yes,

lock the scene.

---

# Author Authority

The author owns:

* the story
* the characters
* the philosophy
* the emotions

AI owns:

* organization
* clarity
* consistency
* continuity

Whenever conflict exists,

the author's intent always wins.

---

# Final Rule

The AI is not writing the novel.

The AI is helping the author express the novel already existing in his mind.

Protect the author's voice before improving the author's writing.

Authenticity is always more important than perfection.
